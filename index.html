Правила ведения проекта

  • Работать на русском языке.
  • Технические имена БД, API, файлов и классов — English.
  • PostgreSQL-only. SQLite не использовать.
  • Изменения схемы БД — только через Alembic.
  • main всегда должен оставаться рабочим.
  • Новую версию или fix не создавать без явной команды пользователя.
  • Перед новой версией сначала показать полный список изменений.
  • Публичный dashboard обновлять при каждой новой версии или fix проекта.
  • Публичный dashboard использовать как журнал работы с датой и временем выполнения.
