Правила

  • Работать на русском языке.
  • Решать проблемы по одному шагу: один шаг → проверка результата → следующий шаг.
  • Технические имена БД, API и файлов — English.
  • Бизнес-термины, документация и UI — русский.
  • PostgreSQL-only. SQLite не использовать.
  • Изменения схемы БД — только через Alembic.
  • main всегда должен оставаться рабочим.
  • Новую версию или fix не создавать без явной команды пользователя.
  • Перед новой версией сначала показать полный список изменений.
  • Интерфейс, меню, context actions, свойства, отчёты и demo flags проектировать через PostgreSQL metadata, а не жёстко зашивать в React.
  • Публичный dashboard и docs/project-journal.md обновлять при новой версии/fix, вечернем отчёте, утреннем плане и важной проверке.
