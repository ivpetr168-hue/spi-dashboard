## 2026-07-01 12:00 Asia/Magadan — оценена достаточность PostgreSQL-данных для реальных отчётов по малой выборке

**Статус:** выполнено. Код не изменялся. Ветка не создавалась. Новая версия не выпускалась.

Вопрос пользователя: достаточно ли текущих данных в PostgreSQL для формирования реальных полных отчётов по небольшой выборке приборов.

Ответ:
- для `Instrument Specification` по малой выборке данных достаточно, особенно по прибору `RURN-NC-QAY52174 -FIT 01`;
- для `Process Data` данных достаточно для реального отчёта по имеющимся строкам;
- для простого `I/O Map` данных достаточно для ограниченной реальной выборки;
- для простого `I/O Tag Assignment` данных достаточно для ограниченной реальной выборки;
- для простого `Cable / Wiring Schedule` данных достаточно по части кабелей, пар/жил и вероятных клемм;
- для полноценного `Point-to-Point Wiring Diagram`, SmartLoop, Loop Signal Diagram и Tag Signal Diagram данных пока недостаточно, если требовать инженерно точную графическую схему на уровне SPI.

Важное уточнение:
- можно формировать реальные отчёты по текущей PostgreSQL-выборке;
- нельзя обещать полное 1:1 повторение всех SPI-отчётов, потому что часть полей SPI отсутствует или хранится пока в JSON/демо-структурах;
- правильная цель `v0.8.5`: сделать реальные отчёты по имеющимся данным, с пустыми/прочеркнутыми полями там, где данные ещё не загружены из Oracle SPI.

Что есть в текущей PostgreSQL-модели:
- структура Plant / Area / Unit / Loop / Instrument;
- приборы FIT/PIT/TIT по `RURN-NC-QAY52174`;
- спецификации с JSON-полями;
- process data;
- документы и ревизии;
- панели, шкафы, terminal strips, terminals;
- кабели, cable sets, wires;
- I/O rack / slot / apparatus / channel;
- control system tags;
- source trace через `source_objects` и `source_object_links`.

Что можно делать в `v0.8.5`:
1. Реальный `Instrument Specification` по FIT/PIT/TIT, с лучшей полнотой по FIT.
2. Реальный `Process Data` report по текущим строкам.
3. Простой реальный `I/O Map` по channel/control system tag/rack/slot/card.
4. Простой реальный `I/O Tag Assignment` по tag/channel/terminal strip/terminal.
5. Простой `Cable / Wiring Schedule` по cable/cable_set/wire/probable_terminal.

Что не делать в `v0.8.5`:
- полноценную графическую генерацию SmartLoop;
- полноценный Point-to-Point Wiring Diagram;
- Loop Signal Diagram / Tag Signal Diagram;
- полное повторение SPI title block и всех revision blocks, если данные отсутствуют.

Решение:
- `v0.8.5` можно утверждать и начинать;
- Oracle дообследование вести параллельно, но не блокировать `v0.8.5`.
