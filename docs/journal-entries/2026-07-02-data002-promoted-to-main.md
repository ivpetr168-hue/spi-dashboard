## Текущее правило

С этого момента `main` считается текущей главной рабочей веткой проекта, но только при сохранении нового обязательного архитектурного правила:

- PostgreSQL — источник истины;
- справочники — отдельные таблицы PostgreSQL;
- связи — через ключи / FK;
- быстрый доступ — через индексы;
- frontend не должен прошивать инженерную модель;
- никаких заплаток, временных подстановок и JSON-костылей вместо нормальной структуры БД;
- импорт/экспорт может меняться, но рабочая модель БД должна оставаться нормализованной и пригодной для промышленной системы.
